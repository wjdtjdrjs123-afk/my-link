# Product Requirements 단일 문서 (PRD) : 마이 링크 (My Link)

## 1. 프로젝트 개요

| 항목 | 내용 |
| :--- | :--- |
| **프로젝트명** | 마이 링크 (My Link) |
| **목적** | 흩어져 있는 개인의 다양한 링크들을 단 하나의 URL로 통합하여 손쉽게 공유할 수 있는 서비스 |
| **대상 사용자** | - 자신의 포트폴리오나 블로그 등을 모아두고 싶은 개발자 및 크리에이터 등 누구나 |
| **핵심 기조** | 불필요한 기능(드래그앤드롭, 활성화 토글 버튼, 모바일 미리보기 등)을 모두 덜어내고 가장 빠르고 단순하게 본질(링크 모음)에 집중하는 간결한 프로필 플랫폼 |

---

## 2. 디자인 및 코어 스택 명세

본 서비스는 최첨단 라이브러리와 서비스 기반 아키텍처를 도입하여 빠르고 유연한 확장성을 도모합니다.

- **Frontend & UI**: Next.js (React) + **shadcn/ui** 기반의 컴포넌트 
  - *복잡한 별도 **테마(Theme) 설정 기능은 전혀 제공하지 않으며**, 통일된 디자인 시스템만 유지합니다.*
- **Auth & Database**: **Firebase (Authentication, Firestore)**
  - *구글 소셜 로그인 전용 연동.* 
  - *데이터베이스 모델링 시 유저 도큐먼트 내에 고유 URL 식별자 겸 사용자 이름인 `displayName` 필드를 필수로 포함하며, 사용자가 생성한 링크 아이템들은 메인 유저 도큐먼트 하위의 **서브 컬렉션(Sub-collection)** 단위로 저장하여 데이터 무결성과 관리를 최적화함.*

---

## 3. 기능 목록

기능은 초기 배포 버전에 개발될 필수 코어 기능과 향후 단일 스펙으로 추가될 분기로 나뉩니다.

### [F-01] 구글 소셜 로그인 연동 (Firebase Auth)
- **개요**: 별도의 이메일/비밀번호 입력 없이 Google 계정을 통해 즉시 회원가입과 로그인을 지원합니다.
- **상세 설명**:
  - Firebase Authentication의 `signInWithPopup` 방식을 이용.
  - 로그인 성공 시, 사용자의 구글 지메일 계정(`example@gmail.com`)에서 앞부분(`example`)을 파싱하여 데이터베이스 유저 정보의 `displayName` 초기값으로 자동 셋업.

### [F-02] 자동 URL 슬러그 배정 및 프로필 관리 (인라인 편집)
- **개요**: 지메일 아이디를 기반으로 생성된 `displayName`을 그대로 개인 페이지 접속 URL로 사용하며, 프로필 정보를 직관적으로 수정합니다.
- **상세 설명**:
  - 접속 URL 규칙: 개별 접속 링크는 `mylink.com/{displayName}` 형식으로 고정되어 일치화됨.
  - **정보 수정**: `displayName`(겸 URL 슬러그)과 짧은 소개글(Bio) 텍스트를 클릭하면 바로 입력창으로 전환되는 **인라인 편집(Inline Editing)** 방식을 사용해 직관적인 수정을 지원. (단축 URL 역할인 `displayName` 수정 시 즉시 URL도 변경됨을 유의)
  - 별도의 프로필 이미지 업로드 기능은 **제공하지 않음**.

### [F-03] 링크 아이템 등록 및 삭제 (CRUD)
- **개요**: 방문자에게 보여줄 외부 URL 링크를 빠르게 등록하고 관리합니다.
- **상세 설명**:
  - 제목(Title)과 목적지 URL 값을 입력받아 생성 후 Firebase 서브 컬렉션에 바로 저장.
  - 별도의 정렬 이동(순서 변경) 및 활성화 토글 기능을 없애 복잡도를 최소화하며, 오로지 등록 및 완전 삭제 방식만 제공.
  - 수정 인터페이스 또한 모달이 아닌 **인라인 편집** 활용 원칙.

### [F-04] 파비콘(Favicon) 자동 추출 기능
- **개요**: 링크를 등록하면 해당 웹사이트의 아이콘을 자동으로 불러와 시각적 요소로 배치합니다.
- **상세 설명**:
  - 사용자가 입력한 링크 도메인을 파싱하여 **구글 제공 API** (`https://www.google.com/s2/favicons?domain={URL}`)를 통해 해당 사이트의 파비콘 이미지를 추출하고 버튼 좌측에 렌더링.

## 4. 추후 도입 예정 기능 (단일 건)

### [O-01] 링크별 클릭 조회수 트래킹 
- **개요**: 각 링크가 클릭된 횟수를 카운트하여 조회수 형태로 시각화해 줍니다.
- **상세 설명**:
  - 복잡한 전체 방문자 통계 기능을 전부 제거하고, 오직 링크 버튼을 누를 때마다 Firestore의 해당 서브 컬렉션 내 도큐먼트의 클릭(Click) 카운트를 1씩 증가시키는 개별 카운트 로직만 나중에 추가 예정.
